Cómo Contribuir¶

Hay muchas formas de contribuir a Jackent y todas son valiosas.

1. Reportar Bugs¶

¿Encontraste un error? Abre un issue con:

  • Descripción clara del problema
  • Pasos para reproducirlo
  • Comportamiento esperado vs actual
  • Capturas de pantalla (si aplica)
  • Versión de Jackent, navegador, OS

Ver: Guía de Reporte de Bugs

2. Proponer Features¶

¿Tienes una idea? Inicia una discusión

  • Describe el feature en detalle
  • Explica el problema que resuelve
  • Proporciona ejemplos de uso
  • Considera alternativas

Ver: Guía de Propuesta de Mejoras

3. Mejorar Documentación¶

La documentación siempre se puede mejorar:

  • Corregir typos o errores
  • Añadir ejemplos o aclaraciones
  • Traducir a otros idiomas
  • Crear tutoriales

Dónde contribuir:

  • Archivos .md en /Doc/docs/
  • README del proyecto
  • Comentarios en el código

4. Escribir Código¶

Pull Requests son bienvenidos:

  • Arreglar bugs reportados
  • Implementar features aprobados
  • Refactorizar código existente
  • Añadir tests

Antes de empezar:

  1. Busca issues con etiqueta good first issue
  2. Comenta en el issue que quieres trabajar en él
  3. Espera confirmación del mantenedor
  4. Sigue el Proceso de Pull Request

5. Ayudar a Otros¶

Participar en la comunidad:

  • Responder preguntas en issues/discussions
  • Revisar Pull Requests
  • Compartir tu experiencia
  • Crear contenido (blogs, videos, etc.)
