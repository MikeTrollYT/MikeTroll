❓ FAQ - Configuración¶

Preguntas frecuentes sobre la configuración de Jackent.

General¶

¿Dónde se guarda la configuración?¶

La configuración se almacena en:

backend/config.json

Dentro del contenedor Docker: 

/app/config.json

Persistencia: El archivo persiste entre reinicios si usas volúmenes Docker.

¿Puedo editar config.json manualmente?¶

Sí, pero no es recomendado mientras el backend está corriendo.

Pasos seguros:

  1. Detener el backend: 

    docker compose stop backend

  2. Editar backend/config.json: 

    {
  "autoDeleteHours": 24,
  "spaceLimitEnabled": true,
  "spaceLimitGB": 100
}

  3. Reiniciar: 

    docker compose start backend

¿Los cambios se aplican inmediatamente?¶

Sí, los cambios desde la interfaz web se aplican al momento.

Excepciones:

  • El monitoreo automático se ejecuta cada 10 segundos
  • Cambios en config.json manual requieren reinicio

Tiempo de Seed¶

¿Qué pasa si configuro 0 horas?¶

Resultado: Los torrents nunca se pausarán automáticamente.

Ventajas:

  • Contribuyes más a la comunidad
  • Mejoras tu ratio en trackers privados

Desventajas:

  • Mayor uso de ancho de banda
  • Torrents siempre activos

El tiempo de seed no funciona¶

Verificar en qBittorrent Web UI:

  1. Abrir: http://localhost:8080
  2. Login: admin / tu contraseña
  3. Herramientas → Opciones → BitTorrent
  4. Verificar "Tiempo máximo de seed"

Causa común: El tiempo se mide desde que el torrent completó su descarga, no desde que se añadió.

¿El seed time se reinicia si pauso manualmente?¶

No, el tiempo de seed se calcula desde completion_on (timestamp de completado).

Si pausas y reanudas:

  • El contador no se reinicia
  • Sigue contando desde la fecha de completado

Auto-borrado¶

¿Se borran los archivos descargados?¶

Sí, cuando se borra un torrent automáticamente:

  1. Se elimina el torrent de qBittorrent
  2. Se borran los archivos de ./Descargas

No hay papelera de reciclaje - Es permanente.

¿Puedo evitar que se borre un torrent específico?¶

No automáticamente, pero puedes:

  1. Opción 1: Mover los archivos fuera de ./Descargas
  2. Opción 2: Configurar autoDeleteHours: 0 (desactivar)
  3. Opción 3: Pausar el torrent manualmente (no se borrará si está pausado)

Los torrents se borran antes de tiempo¶

Causa: El límite de espacio está activado y necesita liberar espacio.

Verificar:

docker compose logs backend | grep "Borrando"

Solución:

  • Aumentar spaceLimitGB
  • Desactivar spaceLimitEnabled
  • Reducir el tamaño de los torrents que descargas

¿Se borran torrents incompletos?¶

No, el auto-borrado solo afecta a:

  • Torrents completados al 100%
  • Torrents en estado "seeding"

Los torrents en descarga nunca se borran automáticamente.

Límite de Espacio¶

¿Cómo calcula el espacio usado?¶

El backend calcula recursivamente el tamaño de:

./Descargas/

Incluye:

  • Todos los archivos descargados
  • Subcarpetas
  • Archivos parciales

Frecuencia: Cada 10 segundos

¿Qué pasa si apago el límite con torrents descargados?¶

Nada - Los torrents existentes permanecen intactos.

Solo se desactiva:

  • La validación al añadir torrents
  • El borrado automático por espacio

¿Puedo configurar límites por torrent?¶

No directamente, pero el backend:

  1. Rechaza torrents individuales > spaceLimitGB
  2. Borra torrents antiguos si el total excede el límite

Workaround: Usar categorías en qBittorrent Web UI.

El límite no se respeta¶

Verificar:

# Ver espacio real usado
docker exec Jackent-qbittorrent du -sh /downloads

# Ver configuración
curl http://localhost:7997/api/settings

Causas comunes:

  1. spaceLimitEnabled: false en config.json
  2. Archivos fuera de /downloads
  3. Permisos incorrectos en carpeta

Espacio en Disco¶

¿Cómo libero espacio manualmente?¶

Opción 1 - Desde Jackent:

  1. Ir a "Descargas"
  2. Seleccionar torrent
  3. Clic "Eliminar"
  4. Marcar "Borrar archivos"

Opción 2 - Desde qBittorrent Web UI:

  1. Abrir http://localhost:8080
  2. Clic derecho en torrent
  3. "Eliminar" → "Eliminar archivos"

Opción 3 - Terminal:

# Borrar todo el contenido
rm -rf ./Descargas/*

# Reiniciar qBittorrent
docker compose restart qbittorrent

¿Puedo cambiar la carpeta de descargas?¶

Sí, editar docker-compose.yml:

qbittorrent:
  volumes:
    - ./qbittorrent-config:/config
    - /ruta/nueva/descargas:/downloads  # Cambiar aquí

Reiniciar: 

docker compose down
docker compose up -d

¿El límite afecta a qBittorrent Web UI?¶

No, el límite solo funciona cuando añades torrents desde Jackent.

Puedes añadir torrents manualmente en qBittorrent, pero:

  • No se validará el límite
  • Se aplicará el auto-borrado si excede

Errores Comunes¶

"El torrent supera el límite configurado"¶

Mensaje completo: 

❌ El torrent (150 GB) supera el límite configurado (100 GB).
   Aumenta el límite en Configuración.

Solución:

  1. Aumentar spaceLimitGB a más de 150 GB
  2. Desactivar spaceLimitEnabled
  3. Buscar un torrent más pequeño

"Espacio insuficiente"¶

Causa: La suma de espacio usado + torrent nuevo > límite.

Solución automática: El backend borra torrents antiguos.

Solución manual:

  1. Borrar torrents manualmente
  2. Aumentar límite
  3. Esperar a que se aplique auto-borrado

config.json no se crea¶

Causa: Permisos del contenedor.

Solución:

# Crear manualmente
docker exec Jackent-backend sh -c 'echo "{\"autoDeleteHours\":0,\"spaceLimitEnabled\":false,\"spaceLimitGB\":50}" > /app/config.json'

# Verificar
docker exec Jackent-backend cat /app/config.json

Mejores Prácticas¶

Configuración Recomendada (Hogar)¶

{
  "autoDeleteHours": 48,
  "spaceLimitEnabled": true,
  "spaceLimitGB": 100
}
  • Seed 48h para buena contribución
  • Límite de 100 GB para uso doméstico
  • Auto-borrado para gestión automática

Configuración Recomendada (Servidor)¶

{
  "autoDeleteHours": 0,
  "spaceLimitEnabled": true,
  "spaceLimitGB": 500
}
  • Sin límite de seed (mejor ratio)
  • Límite alto de espacio
  • No auto-borrar (mantener contenido)

Configuración Recomendada (Temporal)¶

{
  "autoDeleteHours": 12,
  "spaceLimitEnabled": true,
  "spaceLimitGB": 50
}
  • Seed mínimo (12h)
  • Auto-borrado rápido
  • Límite bajo para rotación rápida
