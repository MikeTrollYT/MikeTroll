Entorno de Desarrollo¶

Configura tu entorno local para desarrollar JackBrid.

Requisitos¶

  • Git
  • Node.js 18+
  • Docker y Docker Compose
  • Editor de código (VS Code recomendado)

Clonar el Repositorio¶

# Fork del proyecto en GitHub primero
# Luego clonar tu fork
git clone https://github.com/TU_USUARIO/JackBrid.git
cd JackBrid

# Añadir upstream para mantenerte actualizado
git remote add upstream https://github.com/MikeTrollYT/JackBrid.git

Instalar Dependencias¶

# Backend
cd backend
npm install
cd ..

# Frontend (opcional, no tiene dependencias npm)
# Solo para herramientas de desarrollo

Configurar Variables de Entorno¶

cp .env.example .env
nano .env

JACKETT_URL=http://jackett:9117
JACKETT_API_KEY=tu_api_key
ALLDEBRID_API_KEY=tu_api_key

Levantar el Entorno de Desarrollo¶

Opción 1: Con Docker (recomendado)¶

docker compose up -d

Opción 2: Manual (solo backend)¶

cd backend
npm run dev

Verificar que Funciona¶

# Health check
curl http://localhost:8998/api/health

# Abrir en navegador
open http://localhost:8998

Deberías ver:

  • ✅ Interfaz de JackBrid cargada
  • ✅ Indicadores en verde
  • ✅ Backend respondiendo

Postman¶

Para testing de API:

  1. Descargar Postman
  2. Importar colección de JackBrid (si está disponible)
  3. Configurar variables de entorno

Estructura del Proyecto¶

JackBrid/
├── backend/
│   ├── app.js              # Servidor Express
│   ├── jackettClient.js    # Cliente Jackett
│   ├── alldebridClient.js  # Cliente AllDebrid
│   └── package.json
├── frontend/
│   ├── index.html          # Interfaz principal
│   ├── app.js              # Lógica frontend
│   └── styles.css          # Estilos
├── nginx/
│   └── nginx.conf          # Configuración Nginx
├── Doc/
│   └── docs/               # Documentación
├── docker-compose.yml      # Orquestación Docker
└── README.md

Flujo de Desarrollo¶

1. Crear Rama¶

git checkout -b feat/nueva-funcionalidad

2. Desarrollar¶

  • Hacer cambios
  • Probar localmente
  • Commits frecuentes

3. Probar¶

# Verificar que funciona
curl http://localhost:8998/api/health

# Probar en navegador
# Verificar logs
docker compose logs -f backend

4. Push¶

git push origin feat/nueva-funcionalidad

Troubleshooting¶

Puertos en uso¶

# Cambiar puerto en docker-compose.yml
nginx:
  ports:
    - "8080:80"

Cambios no se reflejan¶

# Reconstruir contenedores
docker compose up -d --build

# Limpiar cache del navegador
Ctrl + Shift + R

Problemas con dependencias¶

# Reinstalar dependencias
cd backend
rm -rf node_modules package-lock.json
npm install
