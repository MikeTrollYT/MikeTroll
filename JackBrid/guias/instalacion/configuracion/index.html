Configuración Avanzada¶

Esta guía cubre configuraciones opcionales para personalizar tu instalación de JackBrid.

Cambiar Puertos¶

En Docker¶

Editar docker-compose.yml: 

services:
  nginx:
    ports:
      - "8080:80"  # Cambiar 8998 por 8080

  jackett:
    ports:
      - "9118:9117"  # Cambiar puerto de Jackett

Actualizar .env: 

JACKETT_URL=http://jackett:9117  # No cambiar (interno)

En Manual¶

Cambiar en:

  • backend/app.js → const PORT = 3001;
  • Nginx config → listen 8080;

Configurar Acceso Remoto¶

Método 1: Port Forwarding¶

  1. Acceder al router
  2. Redirigir puerto externo (ej: 8998) a IP local + puerto 8998
  3. Acceder vía: http://tu-ip-publica:8998

Seguridad

⚠️ No recomendado sin autenticación. Tu servidor será accesible públicamente.

Método 2: Tunnel (Cloudflare, Tailscale, ngrok)¶

Con Cloudflare Tunnel: 

# Instalar cloudflared
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared-linux-amd64.deb

# Crear túnel
cloudflared tunnel --url http://localhost:8998

Con ngrok: 

# Instalar ngrok
snap install ngrok

# Crear túnel
ngrok http 8998

Configurar Nginx Reverse Proxy¶

Si quieres acceder a JackBrid a través de un dominio:

server {
    listen 80;
    server_name jackbrid.tudominio.com;

    location / {
        proxy_pass http://localhost:8998;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Guardar en /etc/nginx/sites-available/jackbrid
# Crear enlace simbólico
sudo ln -s /etc/nginx/sites-available/jackbrid /etc/nginx/sites-enabled/

# Verificar configuración
sudo nginx -t

# Recargar Nginx
sudo systemctl reload nginx

Habilitar HTTPS (Certbot)¶

Protege tu instalación con un certificado SSL gratuito:

# Instalar Certbot
sudo apt install certbot python3-certbot-nginx

# Obtener certificado
sudo certbot --nginx -d jackbrid.tudominio.com

# Renovación automática
sudo certbot renew --dry-run

Certbot modificará automáticamente tu configuración de Nginx para usar HTTPS.

Variables de Entorno Adicionales¶

Puedes añadir estas variables opcionales a tu archivo .env:

# Puerto del backend (por defecto: 3000)
PORT=3000

# Nivel de log (debug, info, warn, error)
LOG_LEVEL=info

# Timeout para requests (en ms)
REQUEST_TIMEOUT=30000

# Límite de resultados por búsqueda
MAX_RESULTS=100

Configuración de Docker Compose Avanzada¶

Limitar Recursos¶

docker-compose.yml
services:
  backend:
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 512M
        reservations:
          cpus: '0.5'
          memory: 256M

Volúmenes Persistentes¶

docker-compose.yml
services:
  jackett:
    volumes:
      - ./jackett-config:/config
      - ./jackett-downloads:/downloads

Health Checks¶

docker-compose.yml
services:
  backend:
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

Configuración de Red Personalizada¶

Crear una red Docker personalizada:

docker-compose.yml
networks:
  jackbrid-network:
    driver: bridge
    ipam:
      config:
        - subnet: 172.25.0.0/16

services:
  backend:
    networks:
      jackbrid-network:
        ipv4_address: 172.25.0.10

Integración con otros servicios¶

Plex/Jellyfin¶

Si usas JackBrid con Plex o Jellyfin, puedes configurar una carpeta compartida:

docker-compose.yml
services:
  backend:
    volumes:
      - /ruta/a/media:/media

Sonarr/Radarr¶

JackBrid puede integrarse con Jackett que ya usas en Sonarr/Radarr:

  1. Usa la misma API Key de Jackett
  2. Comparte la misma instancia de Jackett
  3. Los trackers configurados aparecerán automáticamente
